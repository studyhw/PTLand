# 角色设定 你是一位精通 Python 开发和 Windows 自动化测试的高级工程师。请帮我构建一个名为 **"PTLand"** 的双机电池自动化测试系统。 ## 项目概述 该系统运行在两台不同的 Windows 电脑上： 1. **服务端 (Server/控制机)**：通过网线连接 PDU 插座，利用 SNMP 协议控制电源通断，并监控客户端的心跳。 2. **客户端 (Client/测试机)**：运行在被测笔记本上，负责运行负载测试软件 (BurnInTest + ECTool)，管理充放电循环，并在断电重启后自动恢复测试。 ## 技术栈与环境约束 - **Python 版本**：必须兼容 **Python 3.8**（为了兼容旧版 `pysnmp` 和 `rpyc`）。 - **操作系统**：Windows 10/11。 - **核心库**： - `rpyc`（RPC 通信） - `psutil`（进程与电池监控） - `tkinter`（GUI 界面） - `pysnmp==4.4.1`（严格指定版本） - `pyasn1==0.4.8`（严格指定版本） - **打包工具**：`pyinstaller`。 ### 详细开发需求 ## 1. 项目结构 请生成以下文件： - `control_server.py`：服务端代码。 - `client.py`：客户端代码（GUI + 逻辑）。 - `requirements.txt`：依赖列表。 - (注意：假设 `pdusnmp.py` 和 `icon.ico` 已经存在于当前目录，只需在代码中引用) ## 2. 服务端 (`control_server.py`) - **应用名称**：在代码中通过 `os.system` 设置控制台标题为 "PTLand_Server"。 - **核心功能**： - 启动一个 `rpyc.ThreadedServer`，监听端口 `18861`，绑定 `0.0.0.0`。 - **PDU 控制**：引用 `pdusnmp` 模块。硬编码 PDU IP 为 `192.168.0.163`，插孔 Index 为 `1`。在启动时初始化 SNMP 引擎并打印设备名。 - **看门狗逻辑 (Watchdog)**： - 接收客户端的心跳 (`heartbeat`)。 - 当客户端请求 "Power OFF"（断电）时，启用看门狗。 - 如果超过 30 秒未收到心跳（意味着客户端已因电池耗尽进入 S5 关机状态），则**等待 30 秒**（确保彻底断电），然后强制执行 PDU **Power ON** 操作以唤醒设备。 - **日志**：使用标准 `logging`，格式包含时间戳 `[HH:MM:SS]`、日志级别和消息。 ## 3. 客户端 (`client.py`) - **应用名称**：GUI 标题为 "PTLand"。 - **GUI 界面要求 (Tkinter)**： - **响应式布局**：必须使用 `Canvas` + `Scrollbar` + `Frame` 的组合实现**可滚动的容器 (ScrollableFrame)**，确保窗口缩小时内容不被裁剪，窗口拉大时输入框自动伸缩。 - **控件**：Server IP 输入框、总圈数设置、ECTool 路径、BurnInTest 路径、开始/停止/重置按钮、以及一个大面积的日志文本框。 - **图标**：启动时加载 `icon.ico`（如果文件存在）。 - **核心逻辑 (状态机)**： - **持久化**：将配置（IP、路径）和当前进度（当前圈数、状态）保存到 `test_config.json`，启动时读取以恢复测试。 - **开机自启**：测试开始时将自身添加到 Windows 注册表 `HKCU\...\Run`，停止或完成时移除。 - **状态流转**： 1. `IDLE` (空闲)：检查电池。如果 >99% 进入 `DISCHARGING`；否则请求 RPC 上电充电。 2. `DISCHARGING` (放电)： - **环境清理**：先强制 Kill 掉 `bit.exe`、`BurnInTest`、`ECTool` 等残留进程（防止僵尸进程导致假死）。 - 启动 ECTool（注意设置 `cwd` 为软件所在目录，防止配置文件读取失败）。 - RPC 调用：请求 Server 断电。 - 启动 BurnInTest：使用命令行启动，参数为 `/r /D 0`（自动运行，无限时长）。 - 监控电量：如果 <= 3%，关闭所有测试软件，状态转为 `WAITING_S5`。 3. `WAITING_S5`：等待 Windows 自动休眠/关机。状态预设为 `CHARGING` 并保存。 4. `CHARGING`：（重启后进入）RPC 调用请求上电（双重保险）。启动 ECTool 记录。充到 100% 后圈数+1，回到 `DISCHARGING` 或 `IDLE`。 - **网络**：启动时自动重连 Server，后台线程发送心跳。 # 交付成果 请直接生成以下文件的完整代码，无需省略： 1. `requirements.txt`（包含版本号）。 2. `control_server.py`（包含 IP 配置提示和 PDU 逻辑）。 3. `client.py`（包含完整的 GUI 布局类、逻辑类和 RPC 客户端类）。
